# MCP Workbench — working agreement

## Code comments
Do not add code comments. Write code that explains itself through clear names and structure.

Only add a comment when it is absolutely necessary — i.e. when the code genuinely cannot express the intent on its own and omitting it would invite a real bug or misuse. In that rare case, keep it to a single short line.

Rationale: this code is read by prospective employers during hiring reviews, and AI-style explanatory comments are an immediate tell. Comment-free, self-documenting code is the goal.

## Git
Do not run any part of the git workflow — not `git add`, `commit`, `push`, `branch`, `checkout`, `merge`, `reset`, `tag`, `stash`, or any other git command — unless I explicitly say the exact phrase "run the git sequence".

Until I say that phrase, leave the repository completely untouched.

## No co-authors — ever
Never add a co-author anywhere. No `Co-Authored-By` trailers in commit messages, no co-author lines in changelogs, release notes, PR descriptions, file headers, or any other text. Do not attribute work to Claude/AI anywhere. This is absolute, with no exceptions.
